## Phase 1：驱动层开发 (2天)

### 1.1 SSD1306 OLED 驱动 `ssd1306_i2c.c/h` (3h)
- CubeMX 配置 I2C1（Fast Mode 400kHz）
- 实现 I2C 读写原语
- SSD1306 初始化序列（0xAE→0xAF 等命令序列）
- 实现全屏缓冲（128×64 = 1024字节，1 bit/pixel）
- 实现 `ssd1306_Refresh()` 全屏刷新
- 基本绘图：`SetPixel()`, `Clear()`, `Fill()`

**注意**：SSD1306 页面寻址模式，垂直字节排列（每字节=8像素竖条）

### 1.2 串口命令解析 `cmd_parser.c/h` (2h)
- 基于 USART1 接收中断 + 环形缓冲区
- 支持回车换行（`\r\n`）结尾的文本命令
- 命令格式：`命令名 [参数]`
- 提供 `CmdParser_GetCommand()` 给主循环轮询

**串口命令定义**：
| 命令 | 功能 | 说明 |
|------|------|------|
| `touch` / `t` | 摸头交互 | 触发 SURPRISE 反应 |
| `feed` / `f` | 喂食交互 | 触发 HAPPY 反应 |
| `status` / `s` | 查看状态 | 打印温度、光照、情绪 |
| `help` / `h` | 帮助 | 打印所有命令 |

### 1.3 ADC 传感器驱动 `sensor.c/h` (3h)
- CubeMX 配置 ADC1：CH0(PA0)=LDR, CH1(PA1)=Thermistor
- 启用 DMA 循环采集（双通道）
- 软件均值滤波（滑动窗口，窗口大小=8）
- **LDR 校准**：暗(<500 lux) / 正常 / 亮(>2000 lux) 三档（需要实际测试定阈值）
- **热敏校准**：冷(<18°C) / 舒适 / 热(>30°C) 三档（NTC 查表法或公式拟合）
- 提供 `Sensor_GetLightLevel()` → `LIGHT_DARK / LIGHT_NORMAL / LIGHT_BRIGHT`
- 提供 `Sensor_GetTempLevel()` → `TEMP_COLD / TEMP_COMFORT / TEMP_HOT`

### 1.4 W25Q Flash 驱动 `w25q_spi.c/h` (3h)
- CubeMX 配置 SPI1（Mode 0, CPOL=0 CPHA=0, 18MHz）
- 实现 SPI 收发原语
- W25Q 命令集：Read JEDEC ID, Read Status, Write Enable, Sector Erase(4KB), Page Program(256B), Read Data
- 实现 `W25Q_Init()`, `W25Q_Read()`, `W25Q_Write()`, `W25Q_EraseSector()`
- 上层封装：`Flash_WritePage()`, `Flash_ReadPage()`

### 1.5 无源蜂鸣器驱动 `buzzer.c/h` (1.5h)
- CubeMX 配置 TIM2 CH1 PWM 输出
- 频率映射表（C4~C6 音阶）
- 实现 `Buzzer_PlayTone(freq, duration_ms)`
- 实现 `Buzzer_PlayMelody(notes[], len)` 非阻塞播放（定时器中断驱动）

### 1.6 调试日志 `debug.c/h` (0.5h)
- USART1 printf 重定向（已有）
- 分级日志宏：`LOG_DBG()`, `LOG_INFO()`, `LOG_WARN()`, `LOG_ERR()`
- 编译开关控制日志级别

## Phase 2：中间件层开发 (1.5天)

### 2.1 图形引擎 `gfx_engine.c/h` (3h)
- 依赖 SSD1306 全屏缓冲
- 实现：`GFX_DrawPixel()`, `GFX_DrawLine()`, `GFX_DrawRect()`, `GFX_DrawBitmap()`
- `GFX_DrawBitmap(x, y, w, h, *data)`：核心函数，将外部点阵数据绘制到缓冲
- `GFX_DrawChar()`, `GFX_DrawString()`：6×8 或 5×7 ASCII 字体
- `GFX_InvertRect()`：反色矩形（用于菜单选中效果）
- 提供擦除/局部刷新能力

### 2.2 Python 图片转点阵工具 `tools/img2c.py` (2h)
```python
# 输入：PNG图片序列（如 dog_happy_00.png ~ dog_happy_03.png）
# 输出：pet_sprites.h / pet_sprites.c
# 格式：const uint8_t sprite_dog_happy[4][1024] = { ... };
#        const AnimFrame anim_happy[4] = { {sprite_dog_happy[0], 200ms}, ... };
```
- 将 PNG 转为 1-bit 黑白点阵（阈值=128）
- 按 SSD1306 垂直字节格式排列（每字节=8垂直像素）
- 生成 C 数组 + 帧描述结构体

### 2.3 动画播放器 `anim_player.c/h` (2h)
```c
typedef struct {
    const uint8_t *bitmap;  // 指向帧数据
    uint16_t duration_ms;   // 帧持续时间
} AnimFrame;

typedef struct {
    const AnimFrame *frames;
    uint8_t frame_count;
    uint8_t loop;           // 是否循环
} Animation;

typedef struct {
    Animation *current_anim;
    uint8_t  current_frame;
    uint32_t last_tick;
    uint8_t  playing;
} AnimPlayer;
```
- `AnimPlayer_Play(anim)`：开始播放动画
- `AnimPlayer_Update()`：在主循环中调用，检查时间切换帧
- `AnimPlayer_GetCurrentBitmap()`：获取当前帧数据供渲染

### 2.4 传感器数据映射 `sensor_map.c/h` (1h)
- 组合 LDR + 温度 → 综合环境评分
- 例如：暗+冷=1（差）、亮+舒适=5（好）
- 提供 `Env_GetComfortLevel()` 给 FSM 做状态转换判断

### 2.5 Flash 存储管理 `flash_store.c/h` (2h)
- W25Q 8Mbit 分区规划：
  ```
  0x000000 - 0x00FFFF (64KB) : 系统配置区（阈值参数）
  0x010000 - 0x0FFFFF (960KB): 动画资源区（点阵数据）
  0x100000 - 0x100FFF (4KB)  : 日志/预留
  ```
- `Flash_LoadConfig()` / `Flash_SaveConfig()`：读写配置
- `Flash_LoadSprites()`：从 Flash 加载动画资源到 RAM（或按需读取）
- 首次启动写入默认资源（出厂预设）

## Phase 3：应用层开发 (2天)

### 3.1 小狗状态机 `pet_fsm.c/h` (4h)

**重新设计的状态机**（基于环境感知驱动）：

```
                  ┌─────────────────────────────────────┐
                  │           环境感知驱动 FSM            │
                  └─────────────────────────────────────┘

     ┌──────────┐    亮+舒适    ┌──────────┐
     │  SLEEP   │ ──────────▶  │  HAPPY   │
     │  (睡觉)   │ ◀────────── │  (开心)   │
     │ 暗,安静   │   暗+舒适    │ 亮,温暖   │
     └──────────┘              └──────────┘
          │  ▲                      │  ▲
    长时间无交互                串口交互    │ 环境持续差
          │  │                      │  │
          ▼  │                      ▼  │
     ┌──────────┐    热+亮      ┌──────────┐
     │  IDLE    │ ──────────▶  │  ANNOYED  │
     │  (待机)   │ ◀────────── │  (烦躁)   │
     │ 舒适环境  │   舒适恢复    │ 太热/太亮  │
     └──────────┘              └──────────┘
```

**5个情绪状态**（对齐规格书）：
| 状态 | 触发条件 | 动画表现 | 蜂鸣器 |
|------|---------|---------|--------|
| HAPPY | 亮+舒适温度 | 摇尾巴、笑脸、蹦跳 | 欢快短音 |
| SLEEP | 暗光环境 + 持续30s | 闭眼、呼吸起伏、Zzz | 无 |
| ANNOYED | 过热(>32°C)或过亮 | 皱眉、抖腿、叹气 | 低沉短音 |
| IDLE | 正常光照+舒适温度 | 眨眼、偶尔摇尾 | 无 |
| SURPRISE | 串口交互触发 | 耳朵竖起、惊讶脸 | 短促提示音 |

**状态转换规则**：
- 状态切换有最小停留时间（3秒），防止频繁抖动
- IDLE 是默认状态，所有恢复路径优先回到 IDLE
- 串口交互是瞬时触发，不影响环境判定

### 3.2 交互处理 `interaction.c/h` (2h)
- 串口命令 → 动作映射
- `touch` / `t`：摸头交互 → 触发 SURPRISE 动画 + 蜂鸣器反馈
- `feed` / `f`：喂食交互 → 触发 HAPPY 动画 + 进食音效
- `status` / `s`：打印状态信息（温度、光照、当前情绪）
- `help` / `h`：打印命令列表
- 交互有冷却时间（1秒），防止连发

### 3.3 音效系统 `audio_fx.c/h` (1.5h)
- 预定义旋律序列：
  - MELODY_HAPPY: C5→E5→G5 上行琶音
  - MELODY_SURPRISE: G5 短促单音
  - MELODY_ANNOYED: C4 低沉短音
  - MELODY_SLEEP: 无
- `AudioFX_Play(melody_id)`：触发非阻塞播放

### 3.4 主循环整合 `main.c` (3h)

```c
int main(void) {
    HAL_Init();
    SystemClock_Config();
    // 依次初始化各外设驱动
    SSD1306_Init();
    W25Q_Init();
    Sensor_Init();
    CmdParser_Init();
    Buzzer_Init();

    // 从Flash加载配置和动画资源
    Flash_LoadConfig(&g_config);
    // 如果首次启动则写入默认资源
    if (!Flash_CheckResources()) {
        Flash_WriteDefaultSprites();
    }

    AnimPlayer_Init();
    PetFSM_Init(STATE_IDLE);

    LOG_INFO("Dog Pet System Started!");

    while (1) {
        uint32_t now = HAL_GetTick();

        // 1. 传感器数据采集 + 环境评估
        Sensor_Update();              // DMA 自动，这里读缓冲
        EnvLevel env = SensorMap_Evaluate();

        // 2. 串口命令处理
        CmdEvent cmd = CmdParser_GetCommand();
        if (cmd != CMD_NONE) {
            Interaction_Handle(cmd, now);
        }

        // 3. 状态机更新（环境+交互 → 状态转移）
        PetFSM_Update(env, cmd, now);

        // 4. 动画播放器更新（帧切换）
        AnimPlayer_Update(now);

        // 5. 渲染
        GFX_ClearBuffer();
        GFX_DrawBitmap(32, 0, 64, 64, AnimPlayer_GetCurrentBitmap());
        GFX_DrawStatusBar(env, PetFSM_GetCurrentState()); // 顶部状态栏
        SSD1306_Refresh();

        // 6. 蜂鸣器更新（非阻塞播放下一音符）
        Buzzer_Update(now);

        // 7. 喂狗 or 低功耗等待（可选）
    }
}
```

**主循环时序**：目标 50fps（20ms 一帧），动画帧切换通常 150-300ms

## Phase 4：集成测试与调优 (1天)

### 4.1 功能测试清单
- [ ] OLED 显示正常（无花屏、无闪烁）
- [ ] 5套表情动画播放流畅（帧率稳定）
- [ ] LDR 环境下状态正确切换（遮光→SLEEP，强光→ANNOYED）
- [ ] 热敏温度变化响应正确（手指捏住升温→ANNOYED）
- [ ] 串口命令响应正确，无漏收/解析错误
- [ ] 蜂鸣器音效正常，不阻塞主循环
- [ ] 掉电重启后配置恢复（Flash 读写正常）

### 4.2 调优
- OLED I2C 刷新速度优化（当前400kHz，最多约10fps全屏，实际够用）
- Flash 擦写寿命保护（配置区写入频率限制）
- 动画帧数据 ROM 占用统计（目标 < 20KB 全部资源）
- 内存占用检查（堆栈使用，避免溢出）

### 4.3 实机演示录制
- 录制 2-3 分钟功能演示视频
- 整理项目 README + 硬件接线图 + 演示 GIF

## 风险与对策

| 风险 | 概率 | 对策 |
|------|------|------|
| OLED 是坏的/不兼容 | 中 | 先用逻辑分析仪抓 I2C 波形确认 |
| 热敏电阻型号未知，B值不匹配 | 高 | 实际测量两点温度拟合 B 值 |
| Flash 焊接不良/型号差异 | 低 | 先读 JEDEC ID 确认芯片 |
| 点阵动画设计工作量大 | 高 | 每套表情先做2帧，后续补充 |
| 蜂鸣器音量太小 | 中 | 加三极管驱动电路 |
