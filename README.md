# 月薪喵 MoonCat v1.0

> 基于 STM32F103C8T6 的环境感知虚拟桌面宠物 · 128×64 OLED 全屏动画 · 5 状态情绪 FSM

月薪喵是一只跑在 0.96 寸 OLED 屏上的裸机嵌入式桌面宠物。它通过光敏、热敏传感器感知环境变化，自动在 5 种情绪状态间切换——每种情绪都有独立的逐帧点阵动画、蜂鸣器音效和 LED 闪烁节奏。通过串口发送单字符指令即可与它互动（摸头、喂食、哄睡），传感器阈值校准参数保存在 SPI Flash 中，掉电不丢失。

---

## 硬件平台

| 模块 | 接口 | 说明 |
|------|------|------|
| STM32F103C8T6 | — | Cortex-M3 内核，72 MHz 主频 |
| 0.96 寸 SSD1306 OLED | I²C（400 kHz） | 128×64 像素，白色，4 针 |
| W25Q 8 Mbit SPI Flash | SPI（18 MHz） | 存储动画资源 + 用户配置 |
| 光敏电阻传感器（LDR） | ADC1 CH0（PA0） | 模拟量输出，检测环境光照 |
| 热敏电阻传感器（NTC） | ADC1 CH1（PA1） | 模拟量输出，检测环境温度 |
| 无源蜂鸣器 | TIM2 CH1 PWM | 音阶可编程，非阻塞播放 |
| CH340 USB 转 TTL | USART1（PA9/PA10） | 调试日志 + 串口指令交互 |

## 五种情绪状态

| 状态 | 触发条件 | 动画表现 | 帧率 |
|------|---------|---------|------|
| **😐 待机 IDLE** | 正常光照 + 舒适温度 | 眨眼、偶尔抖耳朵 | ~7 fps |
| **😊 开心 HAPPY** | 明亮 + 舒适，或收到 `f` 喂食指令 | 摇尾巴、蹦跳 | 10 fps |
| **😴 睡觉 SLEEP** | 暗光环境持续 30 秒以上 | 闭眼、呼吸起伏、飘 Zzz… | ~3 fps |
| **😲 惊讶 SURPRISE** | 收到 `t` 摸头指令 | 竖耳朵、惊讶脸 | ~12 fps |
| **🥵 烦躁 HOT** | 温度超阈值或环境过亮 | 皱眉、喘气、抖腿 | ~12 fps |

状态切换由有限状态机驱动，设有 3 秒最小驻留时间防止抖动。串口交互指令会瞬时覆盖环境驱动的状态，交互结束后自动恢复环境判定。

### 状态转换图

```
                  亮+舒适          ┌──────────┐
            ┌──────────────────▶  │  HAPPY   │
            │                     │  开心    │
            │    喂食/舒适恢复     └────┬─────┘
            │                          │
    ┌───────┴──────┐            ┌──────┴──────┐
    │    IDLE      │            │  SURPRISE   │
    │    待机       │──摸头(t)──▶│   惊讶      │
    │   (默认状态)   │            │ (瞬时触发)   │
    └───────┬──────┘            └─────────────┘
            │
            │ 暗光 30s+          ┌──────────┐
            └──────────────────▶ │  SLEEP   │
            │                     │  睡觉    │
            │  过热 / 过亮        └──────────┘
            └──────────────────▶ ┌──────────┐
                                  │   HOT    │
                                  │  烦躁    │
                                  └──────────┘
```

## 核心功能

- **全屏逐帧动画渲染** — 每种情绪配备 4–6 帧手绘 128×64 全屏点阵画面，非字符拼接，真正像素级表情
- **环境感知情绪驱动** — 光敏 + 热敏双传感器 → 综合舒适度评分 → FSM 自动切换情绪状态
- **非阻塞动画与音频** — 帧调度器按情绪设定不同帧率；蜂鸣器旋律播放不阻塞主循环
- **掉电配置保存** — 传感器阈值参数（光照/温度舒适区间）写入 W25Q Flash，上电自动加载
- **单字符串口指令** — `t` `f` `z` `s` `w` `r` `=` `-` `h`，无需回车，即按即响应
- **情绪联动指示灯** — 板载 PC13 LED 闪烁频率随情绪变化：睡觉最慢（~0.7 Hz），烦躁最快（~12 Hz）
- **5 秒状态心跳** — 周期性通过串口输出当前情绪、动画槽位、LDR/ NTC 原始值

## 串口指令表

| 按键 | 功能 | 说明 |
|------|------|------|
| `t` | 摸头 | 触发 SURPRISE 惊讶动画 + 提示音 |
| `f` | 喂食 | 触发 HAPPY 开心动画 + 进食音效 |
| `z` | 强制睡觉 | 调试用，强制进入 SLEEP 状态 |
| `s` | 查看状态 | 立即打印当前情绪、传感器数值 |
| `w` | 保存配置 | 将当前阈值写入 Flash，掉电保留 |
| `r` | 恢复默认 | 擦除 Flash 配置，恢复出厂默认阈值 |
| `=` | 调高热点 | NTC 热阈值 +50（降低灵敏度） |
| `-` | 调低热点 | NTC 热阈值 -50（提高灵敏度） |
| `h` | 帮助 | 打印全部指令列表 |

## 固件架构

```
应用层:      main.c（主循环调度，100 Hz 时基）
中间件层:    pet_fsm（FSM 情绪引擎）  gfx_engine（图形原语）
驱动层:      ssd1306_i2c  w25q_spi  sensor（ADC+DMA）  buzzer（TIM PWM）
HAL层:       STM32CubeF1（GPIO / I²C / SPI / ADC / TIM / USART）
```

全部驱动基于 STM32 HAL 库手写实现，FSM、动画播放器、Flash 配置管理均为独立模块。裸机前后台架构，无 RTOS。

---

## 开发环境与编译

- **IDE**: Keil MDK-ARM v5，启用 MicroLIB
- **代码生成**: STM32CubeMX（工程文件 `stm32dog.ioc`）
- **源码目录**: `Core/Src/`（源文件）、`Core/Inc/`（头文件）
- **核心代码量**: 约 1,500–2,000 行（不含 HAL 库）
- **点阵数据**: 5 套动画 × 4–6 帧 × 1024 字节 ≈ 25 KB，存储于 ROM
